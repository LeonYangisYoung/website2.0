Process Documentation

This page explains the full process behind designing and developing my personal portfolio website for CCT260 Project 2. It includes planning, information architecture, visual decisions, wireframes, coding choices, and accessibility considerations following WebAIM and WCAG 2.0 guidelines.

1. Planning & Information Architecture

I began the project by defining the core structure of the site based on the assignment requirements and common portfolio navigation patterns. I decided on five main pages: Home, Portfolio, Resume, Contact, and Process Documentation. This structure allows users to quickly understand who I am, view my work, access my resume, and learn about my design process.

Before coding, I drafted a simple information architecture (IA) diagram and outlined the content for each page. The Home page introduces me, the Portfolio showcases selected work, the Resume provides background and skills, the Contact page offers a form and map, and this Process page documents the entire project.

Wireframes

I created low-fidelity wireframes to explore layout decisions before writing any HTML or CSS. These sketches helped me maintain visual consistency and plan spacing, hierarchy, and navigation flow.

2. Visual Design Decisions

My visual design aims to be simple, clean, and easy to read. I selected a neutral background, a strong accent color for buttons, and a soft shadow system to create depth without overwhelming the layout. The typography is consistent across pages, using clear headings and readable body text.

I applied a card-based layout for project previews and wireframes. This approach helps organize content into clear blocks, making the site more scannable and user-friendly.

3. Development & Coding Choices

The website was built using CSS HTML , I separated styling into an external CSS file to keep the code clean and maintainable. Images and videos are stored in dedicated images/ and media/ subfolders for better organization.

  • Included comments in both HTML and CSS for clarity.
  • Ensured navigation works across all pages consistently.

4. Accessibility & Usability

Accessibility was an important part of the design process.

Perceivable

  • All images include meaningful alt text for screen readers.
  • Text and background colors have sufficient contrast for readability.

Operable

  • Navigation is simple

Understandable

  • Consistent layout patterns help users predict how pages behave.
  • Simple, clear language is used for headings, labels, and buttons.
